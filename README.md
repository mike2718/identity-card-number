## 程序设计的理念

对于输入的身份证号，尽可能详尽的给出分析，列出可靠的和不可靠的个人信息，以便给使用者判断真假的提示。

程序运行上尽可能依照标准GB 11643-1999。

## 写程序的动机

由于自己和乙方签协议时，乙方给出了有问题的身份证号，导致自己提起诉讼时引起了各种麻烦，
于是想到做一个判断身份证号真伪的程序。
